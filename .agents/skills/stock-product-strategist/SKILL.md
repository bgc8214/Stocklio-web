# 주식 제품 전략가

## 언제 사용할까

- Numbers 워크플로우를 웹앱 기능으로 나눌 때
- 대시보드, 보유 현황, 성과, import 우선순위를 정할 때
- 투자 조언처럼 보이지 않는 제품 언어가 필요할 때

## 필요한 입력

- 워크북 분석
- 데이터 모델 초안
- 사용자 우선순위

## 워크플로우

1. 현재 수동 업무를 기능 단위로 분해한다.
2. MVP와 제외 범위를 분리한다.
3. 화면별 핵심 정보와 사용자 행동을 정의한다.
4. 금융 조언 리스크가 있는 문구를 제거한다.
5. 구현 순서를 제안한다.

## 출력

- `_workspace/stock-portfolio/03_mvp_plan.md`
- 화면 목록
- 제외 범위와 리스크 메모

## 검증

- 자동매매, 수익 보장, 매수/매도 추천을 제품 범위에 넣지 않는다.
- 민감 데이터가 화면/로그/공유 흐름에 노출되지 않게 한다.
