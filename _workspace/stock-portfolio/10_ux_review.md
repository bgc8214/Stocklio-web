# UX 리뷰: 입력과 수정 흐름

## 문제

예수금, 입출금, 보유 종목 추가 폼에서 투자자와 계좌를 매번 직접 입력해야 했다. 이미 포트폴리오에 존재하는 계좌가 있는데도 텍스트를 다시 치게 만들면 오타로 계좌가 중복되고, 데이터 검증도 어려워진다. 기존 항목 수정 흐름도 삭제 후 재입력에 가까워 실제 관리 도구로 쓰기 어렵다.

## 하네스 기준 결정

- 반복 사용되는 도메인 객체는 텍스트 입력보다 선택 컨트롤을 우선한다.
- `투자자 + 계좌`는 하나의 선택 단위로 다룬다.
- 기존 레코드는 목록에서 바로 수정할 수 있어야 한다.
- 수정은 기존 폼을 재사용하되, 버튼 문구와 취소 버튼으로 모드를 분명히 표시한다.
- 새 계좌 생성은 별도 계좌 관리 흐름으로 분리하는 것이 제품 레벨에 맞다.

## 반영

- 보유 종목 추가 폼: 기존 계좌 선택으로 변경
- 예수금 폼: 기존 계좌 선택으로 변경
- 입출금 폼: 기존 계좌 선택으로 변경
- 보유 종목, 예수금, 입출금 목록에 `수정` 버튼 추가
- 수정 클릭 시 기존 값이 폼에 채워지고, 제출 시 기존 항목을 갱신
- 수정 취소 버튼 추가

## 다음 개선

- 계좌 관리 전용 화면에서 새 계좌 생성, 이름 변경, 계좌 유형 변경을 처리한다.
- 종목 상세 편집 모달에서 현재가, 통화, 자동 가격 업데이트 여부를 함께 수정한다.
- 삭제는 즉시 삭제 대신 확인 또는 되돌리기 패턴을 적용한다.
- 선택 박스가 길어질 경우 검색 가능한 combobox로 바꾼다.
