# PRD — Stocklio

> 개인 주식 포트폴리오 웹 관리 서비스

---

## 1. 개요

Stocklio는 Apple Numbers 스프레드시트로 관리하던 개인 주식 포트폴리오를 대체하는 웹 서비스다. 국내·해외 주식을 보유한 개인 투자자가 여러 계좌·투자자 단위로 포트폴리오를 관리하고, 시세 자동 조회·일간 수익 분석·텔레그램 알림까지 한 곳에서 처리할 수 있도록 한다.

---

## 2. 배경 및 문제 정의

| 문제 | 현황 |
|------|------|
| 수동 시세 입력 | Numbers에서 매일 직접 주가를 업데이트해야 함 |
| 환율 계산 분산 | USD 자산의 원화 환산을 수식으로 관리해 오류 위험 있음 |
| 이력 관리 어려움 | 일별 스냅샷·수익률 추이를 스프레드시트로 추적하기 번거로움 |
| 다중 계좌 복잡성 | 투자자·계좌·전략별 분류가 많아 시트 구조가 복잡해짐 |
| 이동성 부족 | 스프레드시트는 모바일 접근성이 떨어짐 |

---

## 3. 목표

- 시세·환율 자동 조회로 수동 입력 제거
- 일간·월간·계좌별 수익률을 자동으로 집계
- 어느 기기에서든 동일한 포트폴리오 상태 조회 가능
- 매일 아침 오늘의 포트폴리오 변동 요약을 텔레그램으로 수신

---

## 4. 범위

### 4.1 포함

- 종목·현금잔고·현금흐름 CRUD
- 계좌·투자자 관리 (다중 계좌, 다중 투자자)
- Yahoo Finance 연동 자동 시세 조회 (미국 주식·ETF, USD/KRW 환율)
- 일별 포트폴리오 스냅샷 자동 저장
- 대시보드 (총 평가금액, 자산 배분, 일간 수익, 수익률 차트)
- 보유 종목 테이블 (필터·정렬, 인라인 수정)
- 계좌 상세 (보유·현금·최근 거래내역)
- 수익률 분석 (일별·월별·계좌별·투자자별)
- 현금흐름 기록 (입금, 출금, 배당, 세금, 수수료)
- 자산 배분 분석 (전략·종목·계좌·투자자·계좌유형별)
- 일일 변동 귀속 분석 (시세 효과 vs. 환율 효과)
- 텔레그램 알림 (일간 다이제스트, 대형 변동 경보)
- Supabase 클라우드 동기화 (Google, Naver, 이메일 로그인)
- Numbers XLSX 가져오기 (미리보기 → 확정 워크플로우)
- JSON 백업·복원

### 4.2 제외

- 실시간 호가 스트리밍 (WebSocket)
- 자동 매매·주문 실행
- 한국 주식 자동 시세 (현재 수동 입력만 지원)
- 세금 신고서 자동 생성
- 소셜 공유·커뮤니티 기능
- 유료 구독 플랜·결제 시스템

## 6. 기능 요구사항

### 6.1 포트폴리오 CRUD

| ID | 요구사항 |
|----|----------|
| F-01 | 종목(Holding)을 추가·수정·삭제할 수 있다 |
| F-02 | 종목에 투자자, 계좌, 전략, 평균단가, 수량을 기록한다 |
| F-03 | 현금잔고(CashBalance)를 계좌·통화별로 관리한다 |
| F-04 | 현금흐름(CashFlow)을 유형(입금/출금/배당/세금/수수료)별로 기록한다 |
| F-05 | 계좌(Account)를 투자자·계좌유형(직투/일반매매/연금/ISA)별로 정의한다 |

### 6.2 시세 자동 조회

| ID | 요구사항 |
|----|----------|
| F-10 | 미국 주식·ETF 시세를 Yahoo Finance에서 자동으로 가져온다 |
| F-11 | USD/KRW 환율을 Yahoo Finance에서 자동으로 가져온다 |
| F-12 | 조회된 시세는 5분, 환율은 1시간 캐시한다 |
| F-13 | 시장 휴장일(NYSE 공휴일)을 인식하여 당일 시세 업데이트를 건너뛴다 |
| F-14 | 자동 시세 대상이 아닌 종목은 수동 가격을 유지한다 |
| F-15 | 시세 조회 성공·실패 이력을 기록한다 |

### 6.3 대시보드

| ID | 요구사항 |
|----|----------|
| F-20 | 총 평가금액(KRW/USD)과 총 손익을 카드 형태로 표시한다 |
| F-21 | 전략·종목·계좌별 자산 배분 도넛 차트를 표시한다 |
| F-22 | 일간 변동(금액·등락률)을 표시한다 |
| F-23 | 수익률 추이 차트(일별 스냅샷 기반)를 표시한다 |
| F-24 | 일간 변동에서 시세 효과와 환율 효과를 분리하여 표시한다 |
| F-25 | 당일 상위 5개 기여 종목을 순위별로 표시한다 |
| F-26 | 대시보드 카드 레이아웃을 드래그앤드롭으로 재배치할 수 있다 |

### 6.4 수익률 분석

| ID | 요구사항 |
|----|----------|
| F-30 | 일별 포트폴리오 스냅샷을 자동으로 저장한다 (서울 시간 기준 아침) |
| F-31 | 월별 수익 요약(순유입·평가금액·손익)을 집계한다 |
| F-32 | 계좌별 스냅샷을 저장하고 계좌별 수익률을 제공한다 |
| F-33 | 최대 낙폭(MDD), 최고 평가금액, 최저 평가금액을 표시한다 |
| F-34 | 기간별(1개월·3개월·6개월·전체) 필터로 성과를 조회한다 |

### 6.5 알림

| ID | 요구사항 |
|----|----------|
| F-40 | 매일 아침 일간 다이제스트를 텔레그램으로 발송한다 |

### 6.7 인증·동기화

## 7. 비기능 요구사항

| 항목 | 요구사항 |
|------|----------|
| **성능** | 시세 포함 대시보드 초기 렌더 ≤ 2초 (캐시 히트 기준) |
| **가용성** | Vercel 플랫폼 기준 월 99.9% 업타임 목표 |
| **확장성** | 1인 사용자 기준 설계; 멀티 테넌트 구조이나 대규모 동시 사용자 최적화 제외 |
| **보안** | Supabase Row-Level Security로 사용자 간 데이터 격리; 서비스 역할 키는 서버사이드에서만 사용 |
| **모바일** | 모바일·태블릿·데스크탑 반응형 지원 |
| **오프라인** | 네트워크 없이 캐시된 데이터로 대시보드 조회 가능 |
| **테스트** | 도메인 로직 단위 테스트, 마이그레이션 테스트, Playwright E2E 스모크 테스트 포함 |

## 8. 사용자 스토리 (핵심)

```
As a 개인 투자자
I want 앱을 열면 내 포트폴리오의 오늘 평가금액과 전일 대비 변동을 바로 볼 수 있다
So that 아침마다 Numbers를 열고 시세를 수동으로 갱신하는 시간을 아낄 수 있다

As a 개인 투자자
I want 오늘 포트폴리오 변동이 시세 때문인지 환율 때문인지 알 수 있다
So that 외환 리스크와 종목 리스크를 구분하여 판단할 수 있다

As a 개인 투자자
I want 매일 아침 텔레그램으로 포트폴리오 요약을 받는다
So that 앱을 직접 열지 않아도 주요 변동을 파악할 수 있다

As a 개인 투자자
I want 계좌·전략·투자자별로 자산 배분을 한눈에 본다
So that 리밸런싱 결정을 빠르게 내릴 수 있다

As a 개인 투자자
I want 월별·연도별 수익률 이력을 차트로 본다
So that 장기 투자 성과를 객관적으로 추적할 수 있다
```

## 9. 기술 아키텍처 요약

| 계층 | 기술 |
|------|------|
| 프론트엔드 | React 19, Vite, Chart.js, Craft.js (실험적) |
| 백엔드 | Node.js 20, Vercel Serverless Functions |
| 로컬 DB | SQLite (Node 내장 DatabaseSync) |
| 클라우드 DB | Supabase PostgreSQL (JSONB state 저장) |
| 인증 | Supabase Auth (Google OAuth, Naver OAuth, 이메일 링크) |
| 시세 | Yahoo Finance (chart endpoint, search endpoint) |
| 알림 | Telegram Bot API |
| 배포 | Vercel (정적 + 서버리스), Cron 서울 07:00 |

## 10. 핵심 데이터 모델

```
State
├── holdings[]          # 보유 종목 (ticker, quantity, averageCost, price, currency)
├── cashBalances[]      # 계좌별 현금 잔고
├── cashFlows[]         # 입출금·배당·세금·수수료 이력
├── accounts[]          # 계좌 정의 (투자자, 유형, 기준통화)
├── portfolioSnapshots[]# 일별 포트폴리오 스냅샷
├── accountSnapshots[]  # 일별 계좌별 스냅샷
└── fxRate              # 현재 USD/KRW 환율
```
