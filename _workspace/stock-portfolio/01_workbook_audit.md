# Numbers 워크북 분석

## 읽기 경로

Numbers 파일은 zip 기반 iWork 패키지였다. 직접 표 데이터를 읽기 어려운 IWA 파일을 포함하고 있어, Numbers 앱으로 `/private/tmp/stock-portfolio-export.xlsx`에 임시 내보내기한 뒤 `openpyxl`로 분석했다.

원본 파일과 변환본은 레포에 복사하지 않았다.

## 시트 구조

내보내기 결과는 총 33개 워크시트였다. `그림` 시트와 내보내기 요약을 제외하면 크게 네 묶음이다.

- 보유 현황: `QQQ`, `S&P500`, `국내주식`
- 자산군 총합: `QQQ 총합`, `S&P500 총합`, `총합 비율`
- 월별 성과: `월별 성과`
- 일별 성과: `3월 성과`, `4월 성과`, `5월 성과`

## 보유 현황 표

주요 보유 표는 같은 헤더를 공유한다.

| 열 | 의미 |
| --- | --- |
| 계좌 | 증권사/연금/IRP/직접투자 계좌 |
| 종목 | ETF, 개별주, 국내주식명 |
| 수량 | 보유 수량 |
| 현재가 | 현재 가격 |
| 평단가 | 평균 매입 단가 |
| 평가금액 | `수량 * 현재가` |
| 매입금액 | `수량 * 평단가` |
| 손익 | `(현재가 - 평단가) * 수량` 또는 `평가금액 - 매입금액` |
| 수익률 | `손익 / 매입금액` |

관찰된 보유 묶음:

- `QQQ - Investor A QQQ`
- `QQQ - Investor B QQQ`
- `QQQ - QQQ 총합`
- `S&P500 - Investor A S&P500`
- `S&P500 - Investor B S&P500`
- `S&P500 - S&P500 총합`
- `국내주식 - Investor A 국내주식`
- `국내주식 - Investor B 국내주식`

## 통화와 환율

- `QQQ - 환율` 시트가 있고 분석 시점 값은 `1450`이었다.
- 해외 직접투자 표는 USD 기준 평가액을 계산한 뒤 환율을 곱해 원화로 환산한다.
- 웹 앱에서는 환율을 단순 상수로 두지 말고 `fx_rate_snapshot`으로 저장해야 한다.

## 집계 방식

현재 워크북은 다음 축으로 집계한다. 아래 값은 웹앱 설계를 위한 구조 예시이며, MVP seed 데이터에는 비식별 이름을 사용한다.

- 투자자: 2명
- 계좌: 연금저축, IRP, 퇴직연금, 토스, 삼성증권, 미래에셋 등
- 자산군/전략: QQQ, S&P500, 국내주식
- 계좌 유형: 연금, 미국 직투, 국내 직투
- 통화: KRW, USD

`총합 비율 - 총합`은 QQQ, S&P500, 국내주식 평가금액을 합산해 전체 비중을 만든다. 실제 총액은 민감정보로 보고 하네스 문서에는 보존하지 않는다.

## 성과 기록 방식

월별 성과:

- 투자자별 계좌 평가금액을 월 단위로 기록한다.
- 합계, 입금, 출금, 월별 증가, 누적 증가, 투자 수익금을 계산한다.

일별 성과:

- 3월, 4월, 5월 시트가 월별로 분리되어 있다.
- 투자자별 계좌 평가금액을 날짜별 column으로 기록한다.
- 자산 Flow 시트에서 투자자별 합산, 증가액, 누적 증가액, 추가입금, 일 수익률, 월 누적을 계산한다.

## 웹 서비스로 옮길 때의 해석

Numbers의 wide-table 날짜 columns는 웹 앱에서는 `portfolio_snapshots` 테이블로 정규화하는 편이 좋다. 날짜가 column이 아니라 row가 되어야 월/일/연도 필터와 차트가 쉬워진다.

보유 현황은 `holdings`와 `price_snapshots`를 분리한다. 현재가는 계속 변하고, 평단가/수량은 거래나 수동 수정으로 바뀐다.

## 주의점

- Numbers 내보내기는 공식 계산이 Excel에서 다를 수 있다고 경고한다. 계산 로직은 웹 앱에서 명시적으로 재구현해야 한다.
- 현재가와 환율은 분석 시점 값이다. 자동 업데이트를 붙이기 전까지는 수동 입력값으로 취급한다.
- 이 프로젝트는 투자 판단을 대신하지 않는 개인 기록 도구다.
