# 로컬 제품화 진행 보고

## 범위

Vercel 배포, 로그인, 다중 사용자 권한은 제외했다. 로컬에서 Numbers 관리 도구를 대체하기 위해 필요한 계좌 중심 관리, 상세 추적, 성과 필터, 현금흐름 유형 확장을 우선 반영했다.

## 완료

- 계좌 레지스트리 추가
- 계좌 생성, 수정, 삭제 UI 추가
- 계좌명 변경 시 보유 종목, 예수금, 현금흐름 참조 동시 갱신
- 계좌 상세 패널 추가
- 보유 종목, 예수금, 현금흐름 입력 시 기존 계좌 선택 사용
- 보유 종목, 예수금, 현금흐름 수정 모드 추가
- 성과 기간 필터 추가
- 현금흐름 유형 확장: 입금, 출금, 배당, 세금, 수수료
- 입출금 보정 성과 계산에서 외부 유입과 투자 성과를 분리
- 마이그레이션 state version 6 및 accounts 포함
- 서버 normalize에도 accounts 포함
- 계좌/예수금 화면의 폼 찌그러짐과 상세 금액 overflow를 CSS 레이아웃 기준으로 수정
- 사용자 지정 대시보드 레이아웃 추가: 카드 드래그 재배치, 우하단 핸들 리사이즈, 숨김/표시, 기본값 복원

## 아직 남은 제품화

- XLSX 업로드 preview를 앱 안에서 직접 처리
- 종목 상세 편집에서 통화, 자동 가격 업데이트, 수동 현재가 수정
- 거래 기반 수량 변경 내역 관리
- 배당/세금/수수료를 예수금 잔액과 자동 연결할지 여부 결정
- 삭제 확인 또는 되돌리기
- 계좌/종목 검색 combobox
- 계산 로직 서버 도메인 모듈 분리와 단위 테스트 확장
- 대시보드 레이아웃 사용자별 프리셋과 더 정교한 모바일 전용 배치

## 검증

- `npm run check`
- `npm run test:migration`
- Playwright desktop/mobile 주요 탭 overflow 검증
- 계좌 관리, 계좌 상세, 성과 필터, 현금흐름 유형 렌더 검증
- desktop/tablet/mobile에서 주요 탭의 body overflow와 패널 protrusion 검증
- 대시보드 레이아웃 드래그 재배치, 우하단 핸들 리사이즈, 저장, 숨김, 기본값 복원 검증
