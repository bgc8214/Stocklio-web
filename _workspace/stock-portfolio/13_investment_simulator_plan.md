# 투자 시뮬레이터 계획

## 결론

Stocklio에 `투자 시뮬레이터`를 추가한다. 목적은 사용자가 실제 보유 포트폴리오를 관리하는 것에서 한 단계 나아가, "만약 이 방식으로 투자했다면 지금 얼마가 되었을까"를 빠르게 비교하고 이해하는 것이다.

2026-05-24 추가 결정: 이 기능의 1차 목표는 일반 정적 차트가 아니라 `영상형 실시간 드로잉 차트`다. 처음 구현부터 선이 시간 순서대로 그려지고, 진행 날짜와 평가금액이 같이 움직이며, 마지막 결과에 도달하는 경험을 기준으로 설계한다. 일반 Chart.js 화면을 먼저 만들고 나중에 애니메이션을 덧붙이지 않는다.

첨부 예시의 핵심은 다음이다.

- 일시 투자와 적립식 투자를 비교한다.
- 특정 종목, ETF, 지수, 레버리지 상품을 긴 기간으로 비교한다.
- 원금선과 평가금액선을 함께 보여준다.
- 시간이 흐르며 차트가 그려지는 애니메이션으로 체감도를 높인다.
- 최종 금액, 투자 원금, 수익률, 최대 낙폭을 짧고 강하게 보여준다.

이 기능은 메인 대시보드가 아니라 `성과` 또는 새 메뉴 `시뮬레이터`에 둔다. 대시보드는 내 실제 자산 현황, 시뮬레이터는 가정 기반 비교 도구라서 정보 성격이 다르다.

## 제품 목표

사용자가 다음 질문을 직접 답할 수 있어야 한다.

- 10년 전 QQQ에 1,000만원을 넣었다면 지금 얼마일까?
- 매월 50만원씩 S&P500에 30년 넣었다면 원금과 평가금액 차이는 얼마일까?
- 같은 총투자금이면 몰빵과 적립식 중 어떤 결과가 나왔을까?
- TQQQ 같은 레버리지 상품은 수익은 크지만 낙폭이 얼마나 컸을까?
- 내 실제 포트폴리오가 QQQ, S&P500, SCHD 같은 기준 대비 잘 가고 있을까?
- 원화 기준으로 보면 환율 때문에 결과가 얼마나 달라졌을까?

## 하네스 역할 검토

### 기능 전문가

시뮬레이터는 단순 차트가 아니라 계산 엔진이다. 시나리오 입력, 과거 가격 조회, 현금흐름 생성, 평가금액 계산, 지표 계산이 분리되어야 한다.

핵심 판단:

- MVP는 `일시 투자`, `정액 적립식`, `몰빵 vs 적립식`, `종목 간 비교` 네 가지로 충분하다.
- 세금, 수수료, 배당 재투자, 환율 히스토리는 옵션으로 확장한다.
- 내 실제 포트폴리오와 벤치마크 비교는 2차 기능으로 둔다.

### 데이터/계산 전문가

정확도는 `조정종가(adjusted close)` 사용 여부가 핵심이다. TQQQ, TSLA, AAPL처럼 분할 이력이 있는 종목은 일반 종가로 계산하면 결과가 크게 왜곡된다.

필수 계산 기준:

- 가격 기준: 월말 또는 거래일 종가
- 기본 가격: 조정종가 우선
- 원금선: 누적 투자 원금
- 평가금액선: 보유 수량 * 해당 시점 가격
- 적립식 매수: 매수일이 휴장일이면 다음 거래일 또는 직전 거래일 정책을 고정
- 레버리지 ETF: 실제 과거 가격을 사용하고, 별도 이론 레버리지 계산은 하지 않는다.

주의:

- Yahoo Finance 무료 데이터는 편리하지만 완전한 금융 데이터 공급 계약은 아니다.
- 장기 과거 데이터가 없는 종목은 시작 가능일을 자동 보정해야 한다.
- 환율을 현재 환율로만 환산하면 장기 시뮬레이션 결과가 부정확할 수 있다.

### UX/콘텐츠 전문가

첨부 예시처럼 콘텐츠형 화면과 관리형 화면은 다르게 설계한다. 일반 앱 화면에서는 필터와 설정을 명확하게, 공유/영상형 화면에서는 큰 제목과 강한 차트가 중요하다.

권장 UX:

- 상단에 시나리오 프리셋을 둔다.
- 사용자는 복잡한 옵션을 처음부터 보지 않는다.
- 기본값으로 바로 차트가 나온다.
- 고급 옵션은 접어서 둔다.
- 결과는 `최종 평가금액`, `총투자금`, `수익`, `수익률`, `최대 낙폭`을 한눈에 보여준다.
- 차트는 원금선과 평가금액선을 함께 보여준다.
- 여러 종목 비교에서는 각 종목 끝점 라벨을 표시한다.
- 애니메이션 재생 버튼과 다시 보기 버튼을 제공한다.

### QA/리스크 전문가

이 기능은 투자 조언으로 오해될 수 있다. "과거 성과는 미래 수익을 보장하지 않는다"는 문구가 필요하다. 또한 레버리지 상품은 장기 투자 위험이 크므로 결과와 함께 최대 낙폭을 반드시 보여준다.

필수 검증:

- 시작일보다 늦게 상장한 종목 처리
- 휴장일 매수 정책
- 분할/배당 반영 여부
- 월 적립일이 없는 날짜 처리
- 여러 통화 섞임 처리
- 모바일에서 차트 라벨 겹침 방지
- 큰 숫자 표기: 원, 만원, 억원 자동 축약

## 차트 구현 방식

1차는 `Custom SVG + requestAnimationFrame`을 사용한다.

이유:

- 사용자가 원하는 것은 일반 분석 차트가 아니라 영상처럼 선이 살아서 그려지는 경험이다.
- SVG는 선 길이, 현재 진행 점, 끝점 라벨, 금액 카운트업을 직접 제어하기 쉽다.
- `stroke-dasharray`, `stroke-dashoffset`, `requestAnimationFrame`을 조합하면 처음 날짜부터 마지막 날짜까지 자연스럽게 재생할 수 있다.
- 진행률에 맞춰 현재 날짜, 현재 평가금액, 원금, 수익률을 동시에 갱신할 수 있다.
- 세로형 콘텐츠 카드와 일반 앱 화면을 같은 렌더 엔진으로 공유할 수 있다.

Chart.js는 보조 분석 차트에만 사용한다.

- 상세 표 옆의 정적 분석 차트
- 툴팁 중심의 탐색형 차트
- 기존 성과 화면의 Numbers 스타일 차트

실시간 드로잉 차트는 다음 전용 모듈로 분리한다.

- `src/app/simulator-animated-chart.js`

책임:

- 데이터 포인트를 SVG 좌표로 변환
- 원금선/평가금액선 path 생성
- path 길이 기반 reveal 애니메이션
- 진행 index 계산
- 움직이는 점과 라벨 렌더
- 금액 카운트업
- 재생/일시정지/다시 재생 제어
- 모바일 라벨 축약

### 애니메이션 동작 기준

- 기본 재생 시간: 5초
- 긴 기간 데이터는 월봉으로 downsample
- 진행률 0에서 시작해 마지막 데이터 포인트에서 종료
- 진행 중 상단 숫자는 현재 포인트 기준으로 갱신
- 마지막 10% 구간에서는 속도를 살짝 늦춰 최종 결과를 체감하게 한다.
- 사용자가 스크롤 중이거나 모바일 저전력 환경이면 자동 재생 대신 `재생` 버튼을 보여준다.

- 그 아래: 핵심 입력 3개만 노출
- 차트 우선 표시
- 고급 옵션과 상세 표는 접기
- 애니메이션은 자동 재생하지 않고 `재생` 버튼 제공

## 콘텐츠 문구 프리셋

시뮬레이터 결과를 이해하기 쉬운 문장으로 자동 생성한다.

예시:

- `1,000만원을 테슬라에 투자했다면`
- `매월 50만원씩 S&P500에 30년 투자했다면`
- `1억 투자, 몰빵 vs 적립식`
- `QQQ와 TQQQ, 같은 돈을 넣었다면`
- `내 포트폴리오 vs S&P500`

## 정확도 정책

기본 정책:

- 미국 주식/ETF는 Yahoo Finance 조정종가 기준
- 한국 주식은 Yahoo Finance `.KS`/`.KQ` 조정종가 기준
- 적립일이 휴장일이면 다음 거래일 사용
- 기간 시작 전에 상장하지 않은 종목은 첫 거래일부터 계산
- 결과에 실제 계산 시작일을 표시

표시 문구:

- `조정종가 기준`
- `배당과 분할은 가격 데이터에 반영된 범위 내에서 계산`
- `세금과 수수료는 제외`
- `과거 성과는 미래 수익을 보장하지 않음`

## 구현 단계

### Phase 1: 계산 엔진

- `simulator-core.js` 생성
- 일시 투자 계산
- 정액 적립식 계산
- 몰빵 vs 적립식 계산
- 다중 종목 비교 계산
- 최대 낙폭 계산
- 단위 테스트 추가

완료 기준:

- 고정 fixture로 계산 결과가 재현된다.
- 휴장일/상장일/빈 데이터 테스트가 있다.

### Phase 2: 과거 가격 API

- `/api/yahoo/history` 추가
- Yahoo chart endpoint에서 range/period1/period2/interval 처리
- adjusted close 추출
- 실패 시 명확한 에러 반환

완료 기준:

- QQQ, TQQQ, AAPL, TSLA, 005930.KS 조회가 된다.
- 데이터가 없는 종목은 UI에서 이유를 표시한다.

### Phase 3: 시뮬레이터 화면

- 새 메뉴 `시뮬레이터` 추가
- 프리셋과 입력 폼 추가
- 결과 카드 추가
- SVG 실시간 드로잉 차트 추가
- 재생/일시정지/다시 재생 추가
- 진행 중 날짜와 금액 갱신 추가
- 끝점 라벨과 움직이는 점 추가

완료 기준:

- 사용자가 30초 안에 기본 시뮬레이션을 실행할 수 있다.
- 실행하면 선이 처음부터 마지막 결과까지 실제로 그려진다.
- 애니메이션 진행 중 현재 날짜, 원금, 평가금액이 바뀐다.
- 모바일에서 입력과 차트가 겹치지 않는다.

### Phase 4: 영상형 UX 완성

- 핵심 결과 문장
- 콘텐츠형 세로 카드 레이아웃 초안
- 금액 카운트업 세부 튜닝
- 마지막 결과 강조
- 원금선과 평가금액선 색상 대비
- 큰 제목 자동 생성
- 모바일 재생 버튼과 축약 라벨

완료 기준:

- 첨부 예시처럼 "시간이 지나며 차트가 그려지는" 체감이 난다.
- 결과 화면만 봐도 어떤 조건으로 시뮬레이션했는지 즉시 이해된다.

### Phase 5: 제품 확장

- 내 포트폴리오 vs 벤치마크
- 환율 영향 분해
- 배당 재투자
- 결과 저장
- 결과 공유 이미지

완료 기준:

- 시뮬레이터가 단순 재미 기능이 아니라 포트폴리오 의사결정 보조 화면이 된다.

## 테스트 계획

### 단위 테스트

- 일시 투자: 시작가/종가 기반 평가금액

- 몰빵 vs 적립식: 총투자금 동일성
- 최대 낙폭: peak-to-trough 계산
- 상장일 보정
- 휴장일 보정

### 통합 테스트

- API history 조회
- 시뮬레이터 실행
- 차트 렌더
- 애니메이션 시작/종료 상태
- 다시 재생 상태
- 에러 상태 렌더

### 시각 검증

- 데스크톱 1440px
- 모바일 390px
- 긴 종목명
- 큰 금액
- 2개, 3개, 5개 종목 비교
- 애니메이션 중간 프레임
- 애니메이션 마지막 프레임
- 원금선과 평가금액선 라벨 겹침

## 오픈 결정

구현 전 최종 결정이 필요한 항목:

- 기본 평가 통화: KRW 우선 또는 USD 우선
- 적립 주기 기본값: 매월 또는 매일
- 기본 비교 종목: QQQ/TQQQ/VOO/SCHD 중 어디까지 노출할지
- 시뮬레이터 메뉴 위치: 독립 메뉴 또는 성과 하위 탭
- 콘텐츠형 공유 카드의 이미지 저장이 MVP에 들어갈지, 2차로 갈지

현재 추천:

- 기본 평가 통화: KRW, 단 USD 원본 수익률도 함께 표시
- 적립 주기 기본값: 매월
- 기본 비교 종목: QQQ, TQQQ, VOO, SCHD
- 메뉴 위치: 독립 메뉴 `시뮬레이터`
- 콘텐츠형 화면: MVP 포함
- 이미지 저장/공유: 2차
