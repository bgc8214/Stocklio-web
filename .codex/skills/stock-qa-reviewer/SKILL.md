# 주식 QA 리뷰어

## 언제 사용할까

- 구현 후 계산식과 Numbers 원본의 의미가 맞는지 확인할 때
- import가 잘못된 시트/열을 매핑할 위험을 점검할 때
- 금융 조언처럼 보이는 표현이나 개인정보 노출을 잡을 때

## 필요한 입력

- 워크북 분석
- 데이터 모델
- 구현 코드
- 테스트 결과

## 워크플로우

1. 보유 현황 계산식을 점검한다.
2. 환율 변환과 통화 표기를 점검한다.
3. 성과 계산에서 입금/출금 조정이 분리되는지 확인한다.
4. import 실패/부분 성공 상태를 검토한다.
5. 금융 조언, 수익 보장, 민감 데이터 노출 표현을 차단한다.

## 출력

- `_workspace/stock-portfolio/05_qa_review.md`
- severity별 findings
- 수정 권고와 pass/fix/redo 판정

## 검증

- 발견 사항은 파일/화면/계산식 단위로 재현 가능해야 한다.
- 원본 금융 데이터 전체를 리뷰 산출물에 붙여 넣지 않는다.
