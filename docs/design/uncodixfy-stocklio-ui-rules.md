# Stocklio Uncodixfy UI Rules

## 목적

Uncodixfy 스킬을 Stocklio에 적용하기 위한 프로젝트 전용 UI 기준이다. 원본 스킬의 핵심은 AI가 자주 만드는 과장된 대시보드 UI를 피하고, 실제 사용자가 매일 보는 금융 관리 도구처럼 평범하고 명확하게 만드는 것이다.

Stocklio는 포트폴리오 기록/계산 도구이므로 보기 좋은 장식보다 다음을 우선한다.

- 오늘 자산이 얼마인지 빠르게 확인한다.
- 왜 올랐고 왜 내렸는지 추적한다.
- 보유 종목, 계좌, 예수금, 스냅샷을 실수 없이 수정한다.
- 모바일에서도 값이 잘리지 않고 핵심 행동이 바로 보인다.

## 적용 원칙

### 1. 내부 도구처럼 보이게 한다

- 대시보드 안에 히어로 섹션을 만들지 않는다.
- 장식성 문구, 감성적인 보조 문장, 의미 없는 상태 배지를 줄인다.
- 페이지 상단은 현재 위치, 핵심 상태, 필요한 액션만 둔다.
- 사이드바와 모바일 하단 내비게이션은 화면 구조를 잡는 장치이지 장식 카드가 아니다.

### 2. 카드보다 데이터 위계를 우선한다

- KPI 카드를 기본 반응으로 남발하지 않는다.
- KPI는 사용자가 의사결정에 쓰는 값만 남긴다.
- 같은 중요도의 카드가 여러 개 나열되어 첫 화면을 압도하지 않게 한다.
- 보조 정보는 카드가 아니라 표, 리스트, 보조 텍스트, 접힘 영역으로 낮춘다.

### 3. 금융 데이터는 잘리지 않는다

- 금액, 수량, 수익률은 `font-variant-numeric: tabular-nums` 기준으로 읽히게 한다.
- 큰 금액은 카드 내부에서 말줄임 처리하지 않는다. 공간이 부족하면 단위 축약, 2줄 배치, 툴팁 중 하나를 명시적으로 선택한다.
- 표는 가로 스크롤이 필요한 경우 헤더와 본문 정렬이 유지되어야 한다.
- 모바일에서는 금액을 우선 노출하고, 출처/시간/보조 설명은 아래로 내린다.

### 4. 액션은 사용 맥락 옆에 둔다

- 종목 추가는 필터 줄 끝에 섞지 않는다. 목록 상단의 주요 액션 영역 또는 우측 드로어 진입 버튼으로 분리한다.
- 수정은 하단 별도 카드보다 행 액션, 드로어, 인라인 편집을 우선한다.
- 삭제는 매 행에 항상 붉은 X를 노출하지 않고 더보기 메뉴 안에 둔다.
- 작업 완료, 실패, 저장 중, 가격 갱신 중은 토스트나 버튼 상태로 즉시 피드백한다.

### 5. 표는 스캔 가능해야 한다

- 모든 헤더를 정렬 가능하게 만들지 않는다. 평가금액, 일 영향, 손익, 수익률처럼 사용 빈도가 높은 열만 정렬한다.
- 정렬 상태는 `기본 -> 오름차순 -> 내림차순 -> 기본` 순환으로 동작한다.
- 행 높이는 과하게 키우지 않고, 숫자 열은 오른쪽 정렬한다.
- 필터, 검색, 정렬은 한 줄에 억지로 구겨 넣지 않는다. 화면이 좁으면 필터 버튼 또는 접힘 영역으로 보낸다.

### 6. 색은 의미를 가져야 한다

기존 Stocklio 팔레트를 우선 사용한다.

- 배경: `#f6f7f3`
- 표면: `#ffffff`
- 텍스트: `#111a15`
- 보조 텍스트: `#6a766f`
- 강조: `#1f7a5b`
- 상승/수익: `#c7433d`
- 하락/손실: `#2f67b1`

색상 규칙:

- 상승과 수익은 빨간색, 하락과 손실은 파란색으로 일관한다.
- 초록색은 브랜드/주요 액션/선택 상태에만 사용한다.
- 파란색은 손실 또는 보조 링크처럼 의미가 있는 경우에만 쓴다.
- 차트 색은 구분 목적이어야 하며 장식용 그라데이션은 쓰지 않는다.

### 7. 둥글고 떠 있는 UI를 줄인다

- 카드 반경은 기본 8px, 특수한 이유가 있어도 10px 이내를 목표로 한다.
- 버튼은 8px 내외의 단순한 직사각형 형태를 유지한다.
- 그림자는 최소화하고, 카드 구분은 border와 배경 차이로 해결한다.
- 플로팅 카드, 유리 효과, 과한 블러, 그라데이션 배경은 사용하지 않는다.

## 화면별 적용 기준

### 대시보드

유지할 것:

- 총자산, 일 증감, 주식/예수금, 자산 비중, 변동 원인.
- 가격 기준과 마지막 스냅샷 상태.

줄일 것:

- 첫 화면을 차지하는 가격 기준/성과 기록/저장 상태 카드 묶음.
- 대시보드 편집 같은 낮은 빈도 액션의 상시 노출.
- 모바일 상단의 보조 상태 문구.

추천 구조:

1. 페이지 제목과 사용자 상태.
2. 핵심 요약 2-3개.
3. 오늘 변동 원인.
4. 자산 비중과 주요 계좌.
5. 최근 성과 흐름.

### 보유 종목

유지할 것:

- 상단 요약, 필터, 검색, 주요 정렬, 10개 단위 페이지.
- 종목 추가는 우측 드로어.
- 행별 더보기 메뉴.

개선 기준:

- 티커/종목명 검색 자동완성은 입력 필드 바로 아래에 둔다.
- 추가/수정 폼은 목록과 분리된 드로어에서 처리한다.
- 표의 숫자는 잘리지 않게 최소 너비와 가로 스크롤을 허용한다.
- 모바일은 표 대신 종목 카드 리스트로 전환한다.

### 계좌/예수금

유지할 것:

- 계좌별 총자산, 주식, 예수금, 종목 수.
- 예수금 입력.
- 직접투자 계좌와 연금 계좌 구분.

개선 기준:

- 계좌 목록과 예수금 입력 영역은 같은 행에서 경쟁하지 않게 한다.
- 긴 금액은 축약 표시만 하지 말고 전체 금액을 볼 수 있는 상태를 제공한다.
- 계좌 유형은 `직접투자 계좌`, `연금 계좌` 두 가지를 기본 분류로 쓴다.
- 선택된 계좌의 상세와 구성은 목록 아래 또는 우측 패널에 명확히 연결한다.

### 성과

유지할 것:

- Numbers처럼 누적 수익, 월 누적 수익, 일일 수익을 한 화면에서 비교한다.
- 기간 필터, 월별 요약, 일별 테이블.

개선 기준:

- 차트 툴팁에는 총자산, 일 증감, 투자손익, 입출금 영향을 모두 표시한다.
- 마지막 점에만 값이 보이는 차트는 피하고, hover/focus 시 모든 지점 값을 확인할 수 있게 한다.
- 차트와 표의 날짜 범위가 항상 일치해야 한다.
- 증감 분석과 일별 성과 카드 높이는 내용 때문에 불균형해 보이지 않게 접힘 상태를 명확히 표시한다.

### 자동화/알림

유지할 것:

- 마지막 실행 시각, 다음 실행 예정, Telegram 알림 설정.
- 수동 테스트 전송.

개선 기준:

- 일반 사용자가 이해할 수 없는 API/cron 세부 문구를 기본 노출하지 않는다.
- 실패한 자동화는 원인과 사용자가 할 수 있는 다음 행동을 보여준다.
- 휴장일/주말은 알림 내용에 반영한다.

### 로그인

유지할 것:

- Naver, Google, Email 로그인.

개선 기준:

- 헤더에 버튼 3개를 그대로 노출하지 않는다.
- 로그인이 필요한 경우 별도 로그인 화면 또는 모달에서 제공한다.
- 모바일 인앱 브라우저 제한이 있는 Google 로그인은 설명과 대체 경로를 제공한다.

## 금지 패턴

- 의미 없는 히어로 카피.
- 카드 안의 카드.
- KPI 카드만으로 구성된 첫 화면.
- 과한 둥근 모서리와 pill 버튼 남발.
- 데이터를 채우기 위한 가짜 차트.
- 모든 행에 항상 보이는 삭제 버튼.
- 필터와 생성 액션이 한 줄에 섞여 사용자가 오해하는 구조.
- 모바일에서 데스크톱 표를 그대로 축소하는 구조.
- 금액 말줄임을 기본값으로 쓰는 구조.

## 구현 체크리스트

프론트엔드 변경 전:

- 이 변경이 실제 사용자의 반복 작업을 줄이는가?
- 삭제해도 되는 설명 문구가 있는가?
- 첫 화면에서 가장 중요한 데이터가 먼저 보이는가?
- 모바일에서 같은 행동이 가능한가?

프론트엔드 변경 후:

- 데스크톱 1440px, 1280px에서 값이 잘리지 않는다.
- 모바일 390px, 375px에서 컴포넌트가 겹치지 않는다.
- 수익/손실 색상이 일관된다.
- 버튼 클릭, 저장, 실패, 로딩 상태가 눈에 보인다.
- 표 헤더와 본문 정렬이 맞는다.
- 페이지별 주요 액션이 필터와 혼동되지 않는다.

```text
금융 관리 도구처럼 평범하고 명확하게 만든다.

금액은 절대 이유 없이 잘리지 않게 하고, 모바일에서는 핵심 데이터와 행동을 먼저 보여준다.
```
