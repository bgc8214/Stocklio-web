# 주식 포트폴리오 오케스트레이터

## 언제 사용할까

- 포트폴리오 관리 웹앱의 큰 작업 흐름을 잡을 때
- 워크북 분석, 데이터 모델, 제품 범위, 구현, QA를 이어야 할 때
- 다음 세션이 이어받을 수 있는 결정적 산출물이 필요할 때

## 필요한 입력

- 사용자 요청
- 원본 Numbers/XLSX 위치 또는 분석 요약
- 기존 `_workspace/stock-portfolio/` 산출물
- 구현 코드 상태

## 워크플로우

1. 요청과 현재 산출물 상태를 확인한다.
2. 필요한 다음 역할을 고른다.
3. 입력 파일과 출력 파일을 명시한다.
4. 금융 데이터 민감도와 계산 정확성 리스크를 확인한다.
5. 최종 요약을 `_workspace/stock-portfolio/final/`에 남긴다.

## 출력

- 단계별 핸드오프 업데이트
- 다음 구현 순서
- 남은 리스크와 QA 필요 항목

## 검증

- 원본 금융 데이터가 의도치 않게 git에 들어가지 않았는지 확인한다.
- 각 산출물이 `docs/harness/stock-portfolio/team-spec.md`의 파일 이름과 일치하는지 확인한다.
